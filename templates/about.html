About Secured Image Browser

Develop an image browser embedded with encryption / decryption functions. The encryption and decryption function must be performed automatically once having or opening an image file. The browser is required to support image raw file format, e.g., *.bmp.

Our group use a Javascript app that provides secure file encryption using the AES-256-GCM algorithm from the WebCryptoAPI included in your browser.

How does it work?

Password

An encryption key is derived from the given password using the PBKDF2-SHA512 algorithm.

Key Derivation

PBKDF2 uses a random 128-bit salt and 80,000 iterations to generate a unique 256-bit encryption key.

Encryption

The file is read into memory and encrypted using AES-GCM with the generated key and a 128-bit random nonce.
